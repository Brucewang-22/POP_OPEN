## 其他硬件模块

### 外置触摸传感器（电容触摸开关模块，外置触摸芯片：VCC/GND/GPIO）

重要说明：
- 本触摸模块的 `GPIO` 来自**外置触摸芯片的数字输出脚**（你在 `demo/hardware/touch/TOUCH.md` 里定义的 p1）。
- 这与 **T5 芯片/板级暴露的“touch 测试引脚”**不同：T5 的 touch 引脚仅用于测试/数字信号验证，这里**不使用**。

接线线序表（外置触摸模块 → 主控，共 3 路，可接 3 个触摸模块逐一验证）：

| 通道 | 模块 p1/GPIO（绿线）→ 主控 | 软件映射 | 说明 |
|:--|:--|:--|:--|
| 0 | P26 | `TUYA_GPIO_NUM_26` | 触摸为高电平（active-high），软件下拉 |
| 1 | P22 | `TUYA_GPIO_NUM_22` | 同上 |
| 2 | P23 | `TUYA_GPIO_NUM_23` | 同上 |

每个触摸模块的 p2/VCC（红线）→ 3V3，p3/GND（黑线）→ GND。模块标称 3.3~5V，建议与主控同为 3.3V。

---

原因：
- 逻辑上，为支持多路触摸实验与更丰富的交互输入，仍采用 3 路独立 GPIO 映射与多通道 `touch_sensor` 抽象，在 `tuya_main` 的本地硬件初始化流程中以 50ms 轮询并按 GPIO 打印 `[TOUCH] pin=...` 日志；
- 硬件上，板级音频驱动将 `P27` 作为扬声器使能引脚（`BOARD_SPEAKER_EN_PIN=TUYA_GPIO_NUM_27`），若继续将触摸通道接在 P27 上，会因 GPIO 下拉/电平翻转干扰扬声器开启状态；
- 因此将触摸通道 1/2 迁移至 `P22/P23`，释放 `P27`，避免对扬声器供电/放音行为产生副作用。

### Audio 模块（板载扬声器 + 外置差分输入）

#### 1）板载扬声器功放使能（SPK_EN）

接线线序表（板载功放 EN 控制脚 → 主控）：

| 模块端口/引脚 | 连接到主控引脚 | 软件映射 | 说明 |
|:--|:--|:--|:--|
| SPK_EN | **P27** | `TUYA_GPIO_NUM_27` / `BOARD_SPEAKER_EN_PIN` | 扬声器功放使能，高电平开启、低电平关闭；由板级音频驱动控制，外设无需额外飞线，只需保持引脚配置一致 |

#### 2）外置差分麦克风/音频前级接口（参考）

若使用开发板预留的差分音频输入接口接入外置差分麦克风/前级模块，推荐接法如下（具体丝印名称与管脚顺序以实物板/原理图为准）：

接线线序表（外置差分模块 → 开发板音频接口）：

| 模块端口/引脚 | 连接到主控引脚 | 软件映射 | 说明 |
|:--|:--|:--|:--|
| IN+（差分正端） | 板载音频接口 `AIN_P`/`MIC_IN+` 等差分正端脚 | - | 外置差分麦克风/前级的差分正端输入，需接到开发板音频差分正端管脚，线长尽量短且与 IN- 成对走线 |
| IN-（差分负端） | 板载音频接口 `AIN_N`/`MIC_IN-` 等差分负端脚 | - | 外置差分麦克风/前级的差分负端输入，接到开发板音频差分负端管脚，与 IN+ 等长并行布线以降低噪声 |
| GND | `AGND/GND` | - | 外置模块参考地，与开发板模拟地相连，建议使用屏蔽线或双绞线并将屏蔽层接地 |
| VCC | `3V3_A` / `3V3`（与板上音频供电同电压域） | - | 外置音频前级供电，引脚电压需在模块工作范围内，建议与开发板音频部分共地 |

> 说明：当前 SDK 音频输入在板级注册为单路 MONO（`TKL_AI_0`），无论使用板载咪头还是外置差分前级，只要接入同一 ADC 通道，软件侧都视为一条麦克风通路。多路麦克风或更复杂音频路由需在硬件与驱动层单独扩展。

### IMU 模块（MPU6050，I2C）

模块型号与引脚定义见 `demo/hardware/imu/IMU.md`。通信方式：标准 I2C，主控使用 **I2C0**（T5 硬件 I2C）。

接线线序表（MPU6050 -> 主控）：

| 模块端口/引脚 | 连接到主控引脚 | 软件映射 | 说明 |
|:--|:--|:--|:--|
| VCC | 3V3 | - | 供电 3.3V，建议与主控同电压域 |
| GND | GND | - | 地 |
| SCL | **P20** | `TUYA_IIC0_SCL` / `TUYA_I2C_NUM_0` | I2C0 时钟 |
| SDA | **P21** | `TUYA_IIC0_SDA` / `TUYA_I2C_NUM_0` | I2C0 数据 |
| AD0 | GND | 设备地址 `0x68` | 默认推荐接法；若接 3V3 则地址为 `0x69` |
| INT | 不接（预留） | - | 当前先按轮询方案预留，不接中断线 |
| XDA | 不接 | - | 外部辅助 I2C，当前不使用 |
| XCL | 不接 | - | 外部辅助 I2C，当前不使用 |

### PWM 舵机（MOTO）

模块定义与接口见 `demo/hardware/moto/MOTO.md`。舵机工作电压 3.7V～4.2V，三线：p1 黄线 PWM 信号、p2 红线供电、p3 棕线 GND。

接线线序表（舵机 → 主控）：

| 模块端口/引脚 | 连接到主控引脚 | 软件映射 | 说明 |
|:--|:--|:--|:--|
| p1 黄线（PWM 信号） | **P32** | `TUYA_PWM2` / `TUYA_PWM_NUM_2` | 舵机 1 PWM 信号 |
| p1 黄线（PWM 信号） | **P34** | `TUYA_PWM3` / `TUYA_PWM_NUM_3` | 舵机 2 PWM 信号 |
| p2 红线（供电） | 3.7V～4.2V 电源 | - | 舵机供电，建议与主控共地；若主控仅 3.3V 可外接电池/稳压 |
| p3 棕线（GND） | GND | - | 地 |
| （预留） | **P8** | `TUYA_PWM6` / `TUYA_PWM_NUM_6` | 预留第三路舵机 PWM，当前不接线 |
