# README 生成提示词

用于基于 C 代码生成简洁、高信息密度的工程
README，重点突出调用关系和核心实现逻辑，避免冗余。

------------------------------------------------------------------------

## Prompt

请基于以下 C 代码生成一份工程级 README，用于帮助工程人员快速理解该模块。

### 核心目标

讲清三件事： 1. 模块是干什么的\
2. 代码是怎么跑起来的（调用关系）\
3. 核心实现思路

------------------------------------------------------------------------

### 输出结构

#### 1. 模块概述

-   功能一句话说明\
-   所在层级（驱动 / 中间层 / 应用）

#### 2. 架构与设计

-   模块划分（如初始化 / 控制 / 状态）\
-   核心设计方式（轮询 / 状态机 / 回调 等）

#### 3. 调用流程（重点）

-   用步骤说明调用关系（谁调谁）\
-   分为：
    -   初始化流程\
    -   运行流程

示例：

    init()
     → xxx_init()
     → xxx_config()

    run()
     → xxx_update()
     → xxx_output()

#### 4. 关键接口

-   函数名 + 作用 + 调用时机\
    （不要逐行解释参数）

#### 5. 核心实现逻辑

-   控制逻辑如何实现（如渐进控制 / 限幅 / 状态切换）\
-   关键流程说明（不要复述代码）

#### 6. 注意事项

-   哪些逻辑不能随便改\
-   时序 / 硬件相关约束（如有）

------------------------------------------------------------------------

### 严格限制

-   内容尽量控制在 300\~500 行以内（越短越好）\
-   不重复代码\
-   不解释基础语法\
-   不写背景介绍\
-   信息密度优先\
-   能省则省

------------------------------------------------------------------------

### 风格要求

-   工程文档风格\
-   简洁直接\
-   面向有经验开发者

------------------------------------------------------------------------

### 强约束

优先删除不重要内容，而不是补充说明
