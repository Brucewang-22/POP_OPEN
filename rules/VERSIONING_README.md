# VERSIONING_README

## 目的
用于规范本项目的版本归档与变更记录流程。

## 版本规则
- 版本从 `0.0.1` 开始。
- 每次归档后按顺序递增版本号。
- 建议递增策略：
  - 小改动/修复：`PATCH` +1（如 `0.0.1` -> `0.0.2`）
  - 常规功能新增：`MINOR` +1（如 `0.0.9` -> `0.1.0`）
  - 重大重构/不兼容变更：`MAJOR` +1（如 `0.9.0` -> `1.0.0`）

## 触发条件
当收到用户指令 **“开始归档”** 后，执行一次版本归档并写入变更日志。

## 记录要求
归档时在同级 `CHANGE.log` 中记录，内容需“略详尽”，至少包含：
- 版本号
- 日期时间
- 改动文件范围
- 功能变更点
- 修复项
- 风险/注意事项（如有）

## 日志格式模板
```text
[Version] 0.0.2
[Date] 2026-03-19 15:30:00
[Scope]
- apps/tuya.ai/your_pop_robot/src/...
- apps/tuya.ai/your_pop_robot/README.md
[Changes]
- 新增：...
- 调整：...
- 修复：...
[Notes]
- ...
```
