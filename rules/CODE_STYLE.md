# 代码规范与重构约束说明

## 核心原则

-   功能不变
-   接口不变
-   时序不变
-   提升可读性与可维护性
-   避免过度设计
    使用本规则的代码风格必须统一

## 命名规范

-   宏：全大写 + 下划线（如 MOTOR_PWM_FREQ_HZ）
-   变量/函数：小写 + 下划线（如 motor_set_speed）
-   布尔变量：is\_/has\_/can\_ 开头

## 格式规范

-   4 空格缩进
-   K&R 大括号风格
-   运算符两侧加空格
-   单行不超过 100 字符

## 注释规范

-   注释"为什么"，不是"做什么"
-   说明硬件约束、时序依赖
-   保持简洁准确

## 去冗余规则

-   删除死代码
-   合并重复逻辑
-   提取重复代码为 static 函数
-   消除多余变量

## 魔法数字

-   禁止直接使用
-   必须提取为宏
-   命名包含单位（MS/US/HZ/DEG）

## 嵌入式约束

-   禁止修改：
    -   中断逻辑
    -   时序流程
    -   寄存器操作顺序
-   禁止假设硬件行为

## AI 使用约束

-   功能不变
-   不改接口
-   不影响硬件行为
-   不过度抽象

## 提交前检查

-   [ ] 功能是否未改变
-   [ ] 是否无冗余代码
-   [ ] 命名是否清晰
-   [ ] 注释是否准确
-   [ ] 是否易于维护
